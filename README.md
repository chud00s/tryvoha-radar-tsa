# 🛰️ Tryvoha Radar

**Аналіз часових рядів повітряних тривог в Україні + OSINT-збагачення з Telegram.**

Tryvoha Radar перетворює сирий лог повітряних тривог (2022–2026) на **ситуаційну
обізнаність і раннє попередження**: показує, *коли* і *де* ризик найвищий,
**прогнозує** короткостроковий ризик по кожній області та накладає на офіційні
тривоги **структуровані події з Telegram** (пуски / рух / влучання / збиття),
видобуті за допомогою LLM.

> ⚠️ Проєкт суто **захисний** — для раннього попередження населення та планування.
> Жодного цілевказання чи наступального застосування.

---

## 🎯 Проблема і користь

Повітряні тривоги не випадкові — у них стійкі патерни (час доби, день тижня,
«коридори» прольотів дронів і ракет через сусідні області, хвилі масованих атак).
Tryvoha Radar робить ці патерни вимірюваними й передбачуваними.

**Для кого:**
- 👤 **Широкий споживач** — «який ризик у моїй області, що робити».
- 🛠️ **Оператор-аналітик** — патерни, масовані атаки, несподівані сплески, поширення
  загроз між областями, якість моделі.

## ✨ Що вміє

**Шар 1 — Backbone (Time Series Analysis)**
- Гап-фрі **погодинний ряд** по 25 регіонах (24 області + Київ).
- Патерни: година × день тижня (heatmap), місячний тренд, рейтинг областей.
- **Прогноз ризику** тривоги в наступні `H=6` год по кожній області (gradient
  boosting + ізотонічна калібрація), з **чесним бектестом** проти двох базлайнів.
- **Масовані атаки** — епізоди, коли ≥18/24 областей одночасно під тривогою; плюс
  **«несподівані сплески»** — дні, що відхиляються від очікуваної норми (рання ескалація).
- **Поширення загроз** — для пари сусідніх A→B рахуємо, як часто нова тривога в A
  супроводжується новою тривогою в B упродовж 6 год, із **lift** і запасом часу.

**Шар 2 — Фішка (Agentic AI)**
- LLM (Claude, `claude-haiku-4-5`) зі **structured tool-use** перетворює вільний
  український текст Telegram на типізовані гео-події `{тип, засіб, область, lat/lon, час}`.
- **Працює і без ключа**: детермінований rule-based фолбек + кеш подій у репозиторії.
- Джерела OSINT: офлайн-зразок / JSON-експорт Telegram Desktop / live через Telethon (`collector.py`).

**Шар 3 — UI/UX (Streamlit), три режими**
- **Споживач** — гейдж ризику + «що робити» + карта ризику по областях.
- **Аналітик** — 5 вкладок: карта ризику, патерни, якість моделі, масовані атаки, поширення.
- **Загрози зараз (OSINT)** — folium-карта з клікабельними маркерами дронів/ракет
  (клік → % поширення для дрона / час підльоту для ракети), пульсуючі влучання (червоні)
  й збиття (зелені), стрічка сповіщень.
- Дизайн «blueprint control room»: темна канва, єдиний cornflower-акцент, Inter +
  JetBrains Mono, 8px, без тіней (тема в `.streamlit/config.toml` + CSS у дашборді).

## ⚙️ Конфігурація, ключі та запуск (для свіжого клону)

Tryvoha Radar **повністю працює офлайн без жодного ключа**: у репозиторії закомічені
семпл-дані (`data/raw/sample_messages.json`, `data/raw/ua_oblasts.geojson`) та оброблені
кеші (`data/processed/series.parquet`, `events.parquet`, `risk_now.parquet`,
`model_metrics.json`). Усі змінні середовища — **опціональні**; вони лише розблоковують
повний функціонал. За відсутності ключів застосунок ґрейсфул-фолбекає: rule-based
екстракція + останній знімок історичних даних.

### Середовище
- **Python 3.9+** (локальний `.venv` — 3.9.6). Закомічений `model_metrics.json` відтворює
  таблицю результатів саме під версіями з `requirements.txt` (верхні межі зафіксовано:
  `numpy<2.1`, `scikit-learn<1.7`, `anthropic<1.0`) — інший sklearn може дати числовий дрейф.

### Швидкий старт
```bash
git clone <repo> && cd tryvoha-radar
make setup        # створює .venv + ставить requirements.txt
make app          # дашборд → http://localhost:8501 (працює одразу, без ключів)
```

### Змінні середовища (всі ОПЦІОНАЛЬНІ)
```bash
cp .env.example .env   # .env у .gitignore — ніколи не комітиться
```

| Змінна | Що вмикає | Де зчитується | Де отримати |
|---|---|---|---|
| `ANTHROPIC_API_KEY` | LLM-екстракцію OSINT-подій (`claude-haiku-4-5`) замість regex-правил | `src/ai_extractor.py` | https://console.anthropic.com/ (≈ копійки) |
| `ALERTS_IN_UA_TOKEN` | Реальний поточний стан тривог (інакше snapshot) | `src/live.py` | https://devs.alerts.in.ua/ (безкоштовний) |
| `TELEGRAM_API_ID` | Live OSINT-інжест через Telethon (`python -m src.collector`) | `src/collector.py` | https://my.telegram.org/apps |
| `TELEGRAM_API_HASH` | Пара до `API_ID` | `src/collector.py` | https://my.telegram.org/apps |
| `TELEGRAM_CHANNELS` | Кома-розділені публічні канали моніторингу, напр. `chan1,chan2` | `src/collector.py` | публічні OSINT-канали |

> Усі лоадери читають через `python-dotenv` + `os.getenv`. На Streamlit Cloud додайте ці
> ключі в **Secrets** (TOML) — Streamlit експонує їх і як змінні середовища, тож `os.getenv`
> працює. Без ключів застосунок тихо лишається в snapshot/rule-based режимі.

### Індикатор актуальності
Кольорова крапка + текст у шапці: «Оновлюється в реальному часі» (live, токен alerts.in.ua)
або «Дані станом на <дата> (Київ)» (snapshot, оновлюється раз на добу).

### Команди / CLI
```bash
make data       # перезавантажити CSV alerts + перебудувати погодинну серію
make forecast   # бектест + прогноз ризику "зараз" (модель + калібрування)
make events     # OSINT-екстракція (LLM за наявності ключа, інакше правила)
make test       # юніт-тести
make all        # data → forecast → events → test
make clean      # видалити data/processed/*.parquet + *.json

python -m src.collector --backfill 200   # засіяти OSINT-події з історії каналів
python -m src.collector                  # live-стрім подій (потрібні TELEGRAM_*)
```
> ⚠️ `make data` ПЕРЕзавантажує живий upstream-CSV Vadimkin (датасет зростає), тож
> відтворені метрики відстежують поточний датасет. `make clean` видаляє закомічені кеші —
> після нього спершу запустіть `make data forecast events`.

## ⚖️ Обмеження і чесність

- OSINT-шар у репозиторії — **демонстраційний зразок**. З `ANTHROPIC_API_KEY` і реальним
  експортом/Telethon працює на живих даних.
- Прогноз — імовірнісна **оцінка ризику**, не гарантія; **не замінює офіційні сповіщення**.
- «Поширення» та «% напрямку дрона» — історична аналітика співнастання тривог між
  сусідніми областями, а не реальна траєкторія. Точний курс показуємо лише за наявності
  даних Telegram.
- Крим і Севастополь — українська територія; на картах показані як «немає даних» (тимчасово окуповані).
