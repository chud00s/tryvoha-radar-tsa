# 🛰️ Tryvoha Radar

**Аналіз часових рядів повітряних тривог в Україні + OSINT-збагачення з Telegram.**

Tryvoha Radar перетворює сирий лог повітряних тривог (2022–2026) на **ситуаційну
обізнаність і раннє попередження**: показує, *коли* і *де* ризик найвищий,
**прогнозує** короткостроковий ризик по кожній області та накладає на офіційні
тривоги **структуровані події з Telegram** (пуски / рух / влучання / збиття),
видобуті за допомогою LLM.

> ⚠️ Проєкт суто **захисний** — для раннього попередження населення та планування.
> Жодного цілевказання чи наступального застосування.

---

## 🎯 Проблема і користь

Повітряні тривоги не випадкові — у них стійкі патерни (час доби, день тижня,
«коридори» прольотів дронів і ракет через сусідні області, хвилі масованих атак).
Tryvoha Radar робить ці патерни вимірюваними й передбачуваними.

**Для кого:**
- 👤 **Широкий споживач** — «який ризик у моїй області зараз / найближчі години».
- 🛠️ **Оператор-аналітик** — патерни, аномалії масованих атак, поширення загроз
  між областями, lead-time між OSINT-повідомленням і офіційною тривогою, якість моделі.

## ✨ Що вміє

**Шар 1 — Backbone (Time Series Analysis)**
- Перетворення інтервалів тривог на гап-фрі **погодинний ряд** по 25 регіонах.
- Патерни: година доби × день тижня (heatmap), місячний тренд, рейтинг областей,
  розподіл тривалостей.
- **Прогноз ризику**: ймовірність тривоги в області в наступні `H=6` год
  (gradient boosting + ізотонічна калібрація), з **чесним бектестом** на строго
  пізнішому періоді проти двох базлайнів.
- **Детекція аномалій** — години різкого сплеску одночасної активності багатьох
  областей (сигнатура масованих атак).
- **Поширення загроз** — для пар сусідніх областей рахуємо, наскільки нова тривога
  в A передвіщає тривогу в B упродовж 6 год, із **lift** над випадковим збігом.

**Шар 2 — Фішка (Agentic AI)**
- LLM (Claude) із **structured tool-use** перетворює вільний український текст
  Telegram-каналів на типізовані гео-події `{тип, засіб, локація, область, lat/lon,
  час, впевненість}`.
- **Працює і без ключа**: детермінований rule-based фолбек + кеш подій у репозиторії.
- Джерела OSINT: офлайн-зразок / JSON-експорт Telegram Desktop / live через Telethon.

**Шар 3 — UI/UX (Streamlit)**
- Перемикач **Споживач ⇄ Аналітик**, інтерактивні карти й графіки, українською.
- Дизайн «blueprint control room»: темна канва, єдиний cornflower-акцент, Inter +
  JetBrains Mono, 8px, без тіней (тема в [.streamlit/config.toml](.streamlit/config.toml)
  + CSS у дашборді). Семантика небезпеки (червоний/жовтий/зелений) збережена для гейджа й карти ризику.

## 🤖 AI як основний інженерний інструмент

Цей проєкт побудовано з AI як основним інструментом розробки (повний лог діалогу — у
сабміті). AI використовується і **в продукті**:
- LLM-екстракція подій із неструктурованого тексту (центральна «фішка»);
- structured output через JSON-схему інструмента (надійний парсинг без regex);
- провайдер-агностичний дизайн (Claude за замовчуванням; легко підставити інший).

## 🚀 Запуск

Репозиторій містить **готові кеші** (`data/processed/*`), тож дашборд стартує одразу,
без ключів і без завантаження даних.

```bash
make setup        # venv + залежності
make app          # запустити дашборд → http://localhost:8501
```

Щоб відтворити весь конвеєр із нуля:

```bash
make data         # завантажити тривоги + побудувати погодинний ряд
make forecast     # бектест моделі + прогноз ризику "зараз"
make events       # екстракція OSINT-подій (LLM за наявності ключа, інакше rules)
make test         # юніт-тести
```

Окремі кроки також доступні як модулі: `python -m src.ingest`, `-m src.transform`,
`-m src.forecast --backtest`, `-m src.ai_extractor`.

## 🔴 Live-режим «тривога зараз» (вмикається на деплої)

За замовчуванням блок «тривога зараз» показує **знімок** — останню годину
історичного датасету (бейдж `🕒 snapshot`). Проєкт **повністю готовий до live**:
варто додати токен — і світлофор показуватиме **реальний поточний стан** (бейдж
`🔴 LIVE`), без жодних змін коду.

- Джерело: [alerts.in.ua API](https://devs.alerts.in.ua/) — оновлення за секунди.
- Реалізація: [src/live.py](src/live.py) → дашборд кешує запит на 30с (повага до лімітів API)
  і **тихо відкочується на snapshot**, якщо токена немає або API недоступне.
- Гібрид: live відповідає «що зараз», а історичний датасет + модель — «коли/де ризик
  і що далі».

**Як увімкнути (розробник, на деплої):**
```bash
# отримати безкоштовний некомерційний токен на https://devs.alerts.in.ua/
echo "ALERTS_IN_UA_TOKEN=ваш_токен" >> .env
python -m src.live      # перевірити, що тягне активні тривоги
```

## 🔌 Інші розширення (опційно)

Скопіюйте `.env.example` → `.env` і заповніть:
- `ANTHROPIC_API_KEY` — вмикає LLM-екстракцію (`claude-haiku-4-5`, вартість ≈ копійки).
- `TELEGRAM_API_ID/HASH/CHANNELS` — live-збір OSINT: `python -m src.ai_extractor --live`.

## ⚖️ Обмеження і чесність

- OSINT-шар у репозиторії — **демонстраційний зразок** (синтетичні повідомлення в
  реалістичному стилі). З `ANTHROPIC_API_KEY` і реальним експортом/Telethon працює на живих даних.
- Дані тривог — волонтерські (OSINT), рівень областей; можливі шум і пропуски.
- Прогноз — імовірнісна оцінка ризику, не гарантія; не замінює офіційні сповіщення.
